## 📌 Project Overview

This project develops a robust forecasting system to predict Bitcoin (BTC/USD) closing prices using historical market data. It combines classical time series models, gradient boosting, and deep learning to evaluate and compare multiple forecasting approaches — culminating in an interactive web application powered by real trained model output.

## 🚀 How to Run

### Step 1 — Install Dependencies

```bash
pip install pandas numpy matplotlib seaborn scikit-learn xgboost lightgbm statsmodels tensorflow prophet pmdarima joblib
```

### Step 2 — Run Notebooks in Order

| Step | Notebook                                           | Input                     | Output                    |
| ---- | -------------------------------------------------- | ------------------------- | ------------------------- |
| 1    | `Notebook1_EDA_Preprocessing.ipynb`                | `BTC-USD.csv`             | `BTC_cleaned.csv`         |
| 2    | `Notebook2_FeatureEngineering_ModelTraining.ipynb` | `BTC_cleaned.csv`         | `models/`, `predictions/` |
| 3    | `Notebook3_ModelEvaluation.ipynb`                  | `models/`, `predictions/` | `btc_data.json`           |

> ⚠️ Notebooks **must** be run in order. Each notebook depends on the output of the previous one.

### Step 3 — Launch the Web App

Place `btc_data.json` in the same folder as `BTC_Prediction_WebApp.html`, then:

```bash
# Navigate to the project folder
cd path/to/BTC-Price-Prediction

# Start local server
python -m http.server 8000
```

Open your browser and go to:

```
http://localhost:8000/BTC_Prediction_WebApp.html
```

## 🔧 Feature Engineering

50+ features engineered across 6 categories:

- **Moving Averages** — MA (7, 14, 21, 30, 50, 100, 200), EMA (9, 12, 26, 50)
- **Oscillators** — RSI (14), Stochastic %K & %D, MACD + Signal + Histogram
- **Volatility** — Bollinger Bands (20), Band Width, ATR (14)
- **Lag Features** — Lag 1, 2, 3, 5, 7, 14, 21, 30 days
- **Rolling Statistics** — Std, Min, Max, Skewness over 7/14/30 day windows
- **Date Features** — Day of week, month, quarter, year, month-end flags

## 🔑 Key Findings

1. **Gradient boosting models dominate** — LightGBM and XGBoost outperform all other models by a large margin due to rich feature engineering.
2. **Lag features are most predictive** — Lag_1 (yesterday's price) consistently ranks as the most important feature across all ML models.
3. **BTC price is non-stationary** — First differencing (d=1) is required for ARIMA/SARIMA. ML models handle this natively via lag and return features.
4. **LSTM is competitive** — With proper EarlyStopping (`patience=20`) and learning rate floor (`min_lr=1e-5`), LSTM achieves R²=0.971.
5. **Weak annual seasonality** — BTC shows stronger multi-year cycle patterns than intra-year seasonal patterns.
